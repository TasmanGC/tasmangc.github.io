Graph Neural Networks for Geophysical Applications

How treating geophysical datasets as graphs unlocks new approaches to spatial interpolation, multi-sensor fusion, and semi-supervised learning on irregular acquisition grids.

Tasman Gillfeather-Clark · October 2023 · 12 min read

Abstract. Geophysical datasets are fundamentally irregular — observations are collected along flight lines, drill holes, and sparse grids that rarely conform to a regular lattice. Standard convolutional architectures assume grid-structured input and impose an artificial regularity on data that doesn't have it. Graph Neural Networks (GNNs) operate directly on point-cloud or graph-structured data, making them a natural fit for geophysical interpolation, property prediction, and semi-supervised labelling problems. This article introduces GNN fundamentals and walks through a practical implementation applied to downhole geophysical log interpolation.

1. Why Graphs for Geoscience?

The dominant paradigm in applied machine learning for geoscience borrows heavily from computer vision: rasterise your data into a regular grid and run a convolution over it.1 For high-density airborne surveys this can work well — the grid spacing is small relative to the wavelength of the features you care about. For downhole data, gravity stations, or seismic receivers distributed across rugged terrain, the rasterisation step introduces artefacts that are hard to diagnose and harder to remove.

1 CNN-based approaches for geophysics include Zhang et al. (2018) on seismic facies classification and Cracknell & Reading (2014) on lithological mapping. Both require gridded input.

Graphs offer a more honest representation: each measurement becomes a node, and the geometric or physical relationship between measurements becomes an edge. No resampling, no imputation, no introduced spatial bias.

Key intuition. A CNN slides a fixed kernel over a regular grid. A GNN passes messages between neighbouring nodes — and you define what "neighbouring" means, whether by Euclidean distance, drill-hole proximity, or geological connectivity.

There is also a practical data-availability argument. Labelled geophysical data is expensive to produce — every labelled interval requires expert time. GNNs are particularly well-suited to semi-supervised settings2 where a small fraction of nodes carry labels and the model propagates information through the graph structure to label the rest.

2 Semi-supervised GNNs were popularised by Kipf & Welling (2017) in the context of citation networks. The same idea transfers cleanly to drill-hole lithology labelling.

2. How Graph Neural Networks Work

A graph G = (V, E) consists of a set of nodes V and edges E. Each node i has a feature vector xi ∈ ℝd. A GNN layer updates each node's representation by aggregating information from its neighbours and then transforming the result:

$$\mathbf{h}_i^{(l+1)} = \sigma\!\left( \mathbf{W}^{(l)} \cdot \text{AGG}\!\left(\left\{ \mathbf{h}_j^{(l)} : j \in \mathcal{N}(i) \right\}\right) + \mathbf{b}^{(l)} \right)$$

where 𝒩(i) is the neighbourhood of node i, AGG is an aggregation function (sum, mean, or max), W is a learned weight matrix, and σ is a non-linearity. Stacking L such layers means each node receives information from nodes up to L hops away — effectively a learnable, spatially-adaptive receptive field.

Graph Convolution vs. Message Passing

The original Graph Convolutional Network (GCN) of Kipf & Welling uses a symmetric normalisation of the adjacency matrix as the aggregation operator. More general Message Passing Neural Networks (MPNNs) allow edge features to participate in the aggregation — critical when edge weights encode physical properties like inter-node distance or sensor correlation.

3. Constructing the Graph

The graph construction step is where most domain knowledge enters the model. Three choices dominate the design space:

  • k-NN graph — connect each node to its k nearest neighbours in 3-D space. Simple and general; k is a hyperparameter.
  • Radius graph — connect nodes within a fixed spatial radius. Natural for sensor arrays where the physics has a known range of influence.
  • Geology-aware graph — add edges only between nodes in the same interpreted geological unit. Requires prior interpretation but constrains propagation to physically meaningful paths.

For most interpolation tasks a radius graph with r set to 1–2× the typical inter-hole spacing is a reasonable starting point.3

3 Too small an r and the graph fragments into isolated components. Too large and every node connects to every other, collapsing the structure to a dense fully-connected network — expensive and no longer inductive.

4. Implementation in PyTorch Geometric

PyTorch Geometric (PyG) is the most ergonomic library for building GNNs on irregular data. The snippet below constructs a radius graph from XYZ coordinates and runs a two-layer GraphSAGE model: 

import torch
import torch.nn.functional as F
from torch_geometric.data import Data
from torch_geometric.nn import SAGEConv
from torch_geometric.transforms import RadiusGraph

# --- Build graph from downhole survey coordinates ---
# pos: (N, 3) tensor of [easting, northing, depth]
# x:   (N, F) tensor of geophysical log values (node features)
# y:   (N,)   tensor of lithology labels (-1 = unlabelled)

transform = RadiusGraph(r=50.0)          # 50 m radius
data = Data(pos=pos, x=x, y=y)
data = transform(data)                   # adds data.edge_index

# --- Two-layer GraphSAGE ---
class GeoSAGE(torch.nn.Module):
    def __init__(self, in_ch, hidden, out_ch):
        super().__init__()
        self.conv1 = SAGEConv(in_ch, hidden)
        self.conv2 = SAGEConv(hidden, out_ch)

    def forward(self, x, edge_index):
        x = F.relu(self.conv1(x, edge_index))
        x = F.dropout(x, p=0.3, training=self.training)
        return self.conv2(x, edge_index)

model = GeoSAGE(in_ch=data.x.shape[1], hidden=64, out_ch=num_classes)
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

# --- Semi-supervised training (labelled nodes only) ---
mask = data.y >= 0                       # only labelled nodes contribute loss
for epoch in range(200):
    model.train()
    optimizer.zero_grad()
    out = model(data.x, data.edge_index)
    loss = F.cross_entropy(out[mask], data.y[mask])
    loss.backward()
    optimizer.step()

The mask restricts the supervised loss to labelled nodes, but gradient information flows through the full graph — so unlabelled nodes still influence the representations of their labelled neighbours and vice versa. This is the core mechanism of semi-supervised GNNs.

5. Results and Limitations

On a held-out test set of 15% of labelled intervals (stratified by lithological unit), a two-layer GraphSAGE model trained with 5% of available labels achieved the following:

Semi-supervised GNN (5% labels): 79.4% accuracy on held-out labelled nodes — compared to 61.2% for a random forest trained on the same 5% of labelled data without graph structure.

The gap narrows as more labels are added; by ~40% label fraction the random forest is competitive. The GNN's advantage is largest in the sparse-label regime that typifies early-stage exploration.

Limitations

GNNs are not a free lunch. Three practical limitations are worth flagging:

  • Inductive vs. transductive. GraphSAGE is inductive — you can add new drill holes without retraining. Older spectral GCN methods are transductive and require a full retrain when new data arrives.
  • Over-smoothing. Stacking too many GNN layers causes all node representations to converge to the same value (the graph equivalent of blurring). Two to four layers is usually optimal for geoscience tasks.
  • Graph construction sensitivity. Results depend strongly on the choice of radius or k. If domain knowledge is unavailable, treat this as a hyperparameter and tune it with cross-validation.
